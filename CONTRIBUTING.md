# 🤝 贡献指南

感谢你对这个项目的关注！欢迎提交你发现的优质学习资源。

## 📋 提交资源的格式

### 标准格式

在对应分类的表格中添加一行：

```markdown
| [资源名称](链接) | 📺 视频 | 🇨🇳 中文 | 推荐理由（简短描述为什么这个资源讲得好） |
```

### 资源类型标签

| 标签 | 含义 |
|------|------|
| 📺 | 视频 |
| 📝 | 博客/文章 |
| 📄 | 论文 |
| 💻 | 代码/项目 |
| 📖 | 书籍 |
| 🎧 | 播客/音频 |
| 📱 | 公众号/Newsletter |
| 🔧 | 工具 |

### 语言标签

| 标签 | 含义 |
|------|------|
| 🇨🇳 | 中文 |
| 🇬🇧 | 英文 |
| 🇯🇵 | 日文 |
| 🌐 | 多语言 |

## ✅ 资源收录标准

我们收录的资源应该满足以下条件：

1. **讲解清晰** - 把复杂概念讲得通俗易懂
2. **内容准确** - 没有明显的错误或误导
3. **有独特价值** - 提供了独特的视角或讲解方式
4. **质量稳定** - 对于系列内容，整体质量应该稳定

## 🆕 添加新分类

如果你觉得需要添加新的分类：

1. 在 `categories/` 下创建新目录
2. 创建 `README.md` 文件，参考现有分类的格式
3. 在主 `README.md` 中添加链接

## 💬 提交方式

### 方式一：提交 PR

1. Fork 本仓库
2. 在对应分类中添加资源
3. 提交 Pull Request

### 方式二：提交 Issue

如果你不熟悉 Git 操作，可以直接提交 Issue，包含以下信息：

- 资源名称和链接
- 资源类型（视频/博客/论文等）
- 语言
- 推荐理由
- 建议放在哪个分类下

## ❓ 问题反馈

如果发现：

- 链接失效
- 资源内容有误
- 分类不准确

欢迎提交 Issue 告诉我！
